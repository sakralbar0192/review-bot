## **Повестка первой проектной встречи**

**Дата:** 06.12.2025
**Участники:** Данил, Лев
**Цель:** Согласовать видение проекта, определить MVP, создать дорожную карту и proposal для руководства.  
**Время:** 60-90 минут

### **Часть 1: Контекст и цели (15 мин)**

1. **Введение проблемы** (5 мин)
    
    - Текущий процесс ревью: как работает, какие боли (задержки, несправедливая нагрузка)
        
    - Количественная оценка: сколько времени теряется, сколько MR висит без ревью
        
2. **Цели проекта** (10 мин)
    
    - Бизнес-цель: ускорить delivery за счет предсказуемого процесса ревью
        
    - Техническая цель: создать масштабируемую систему автоматизации code review
        
    - Экспериментальная цель: опробовать DDD/BDD + ИИ в реальном проекте
        
    - Личные цели: выступление с докладом, развитие инженерных навыков
        

### **Часть 2: Определение пользователей и сценариев (20 мин)**

1. **Стейкхолдеры** (5 мин)
    
    - Кто вовлечен: разработчики, тимлиды, менеджеры, руководство
        
    - Их интересы и "боли"
        
2. **Пользовательские сценарии (User Stories)** (15 мин)
    
    - Заполняем шаблон User Story Canvas
        
    - Фокусируемся на сценариях для MVP
        

### **Часть 3: Функциональность и MVP (25 мин)**

1. **Брейншторм фич** (10 мин)
    
    - Все возможные функции без приоритизации
        
    - Записываем в шаблон Feature List
        
2. **Определение MVP** (15 мин)
    
    - Критерий: "Минимальный набор, который решит основную проблему"
        
    - Выбираем 3-5 ключевых фич для первой версии
        
    - Определяем, что оставляем на будущее
        

### **Часть 4: Техническое видение и риски (15 мин)**

1. **Технологический стек** (5 мин)
    
    - Бот: Node.js + библиотека для Slack/Telegram
        
    - БД: PostgreSQL / SQLite для начала
        
    - Интеграции: GitLab/GitHub API
        
2. **Риски и ограничения** (10 мин)
    
    - Технические: доступ к API, безопасность токенов
        
    - Организационные: сопротивление команды, время на разработку
        
    - Юридические: сбор статистики, прозрачность
        

### **Часть 5: План действий и следующий шаг (10 мин)**

1. **Дорожная карта** (5 мин)
    
    - Фазы разработки
        
    - Оценка времени для MVP
        
2. **Следующие шаги** (5 мин)
    
    - Кто что делает до следующей встречи
        
    - Дата следующей синхронизации
